# Visual system

## Communication

- Make the primary journey scannable in under 30 seconds.
- Prefer cards, steps, arrows, timelines, state diagrams, comparisons, and annotated frames over walls of text.
- Use short labels and direct sentences.
- Keep evidence near the claim it supports.
- Use code only when it explains behavior better than prose.

## Meaning

- Observed: neutral or cool accent.
- Reported: secondary neutral accent.
- Inferred: amber or caution accent.
- Proposed: violet or future-state accent.
- Unknown: dashed border or open-question treatment.
- Failure: red only for genuine errors or harm, not decoration.
- Success: green only for verified healthy or completed states.

Always pair color with visible text or an icon.

## Layout

- Use a centered reading width with full-width journey bands where useful.
- Keep cards aligned but allow natural height.
- Collapse multi-column layouts to one column below 720 px.
- Avoid horizontal scrolling at 390 px.
- Use a print layout that removes decorative backgrounds and preserves labels.

## Accessibility

- Use one `h1` and ordered heading levels.
- Include a skip link and semantic landmarks.
- Keep text contrast at WCAG AA targets.
- Make focus styles obvious.
- Add useful alternative text for meaningful images.
- Mark decorative images with empty alternative text.
- Do not put essential information only in an image.
- Respect `prefers-reduced-motion`; static output is the default.

## Restraint

- Use system fonts.
- Use no remote assets.
- Avoid glassmorphism, excessive gradients, ornamental charts, and animation that does not explain state.
- Optimize for credibility and comprehension, not novelty.
