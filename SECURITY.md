# Security policy

Please report suspected vulnerabilities privately through GitHub Security Advisories once the public repository is available. Do not include credentials, private source material, or personally identifiable information in a public issue.

Security-sensitive areas include:

- prompt injection through tickets, logs, screenshots, and source documents;
- secret or personal-data leakage into generated HTML;
- unsafe archive or file-path handling;
- script execution and dependency supply chain;
- external network requests or tracking in generated artifacts;
- HTML injection and unsafe URL schemes.

Until a public advisory channel exists, do not publish proof-of-concept exploit details.
